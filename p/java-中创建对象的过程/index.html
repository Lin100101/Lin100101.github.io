Java 中创建对象的过程

📅 2025-12-04

在 Java 中，创建对象（Object Creation） 是指在堆内存（Heap）中分配一块空间，用于存放某个类的实例（instance），并对其进行初始化。这是面向对象编程的核心操作之一。

我们先看一个简单的例子：

1
2

int a;
Integer b;

这两行代码中，哪一行创建了对象？

答案是：都没有！

虽然 Integer 是 int 的包装类，但 int a; 和 Integer b; 有着本质区别：

  • int a; → 声明了一个基本类型变量，它不是对象；
  • Integer b; → 声明了一个引用类型变量，它可以指向一个 Integer 对象，但此时并未创建任何对象，其值为 null（如果是成员变量）或未初始化（如果是局部变量）。

这就引出了 Java 中两个核心概念：

基本类型 vs 引用类型

Java 的数据类型分为两大类：

类型 说明 示例
基本类型（Primitive Types） 不是对象，直接存储值，效率高 int, double, boolean, char 等
引用类型（Reference Types） 表示对对象的引用（即“指针”） String, Integer, 自定义类如 Person 等

因此：

  • int a; 只是声明了一个用于存储整数值的变量；
  • Integer b; 虽然是引用类型，但仅声明了引用，并未创建对象。

只有引用类型才能指向对象，而要真正拥有一个对象，还需要显式地创建它。

Java 中创建对象的标准流程

当使用 new 关键字创建对象时，JVM 会按以下顺序执行：

检查类是否已加载 → 在堆中分配内存 → 字段设为默认值 → 填充对象头信息 → 调用构造器 <init> 进行显式初始化

具体步骤如下：

步骤 阶段 说明
1 类加载检查 JVM 检查该类是否已被加载；若未加载，则先完成加载、链接（验证、准备、解析）和初始化。
2 分配内存 在堆中为对象分配所需大小的内存空间（大小由所有实例字段决定）。
3 零初始化 将所有实例字段设置为默认值（如 int → 0，Object → null，boolean → false）。
4 设置对象头 填充对象头信息，包括：指向类元数据的指针、哈希码（延迟初始化）、GC 分代年龄、锁状态等。
5 执行构造器 调用 <init> 方法（即你编写的构造函数），执行自定义的初始化逻辑。

完成后，new 表达式返回该对象在堆中的引用。

Integer b; 处于哪个步骤？

它不属于上述任何一个步骤。

因为 Integer b; 只是声明了一个引用变量，并没有触发对象的创建流程。
你可以把它理解为：“我预留了一个‘指针槽位’，但还没指向任何实际对象。”

只有当你写出如下代码时，才会进入对象创建流程：

1
2
3

Integer b = new Integer(10);      // 显式创建新对象（不推荐）
Integer b = Integer.valueOf(10);  // 推荐方式，可能复用缓存
Integer b = 10;                   // 自动装箱，等价于 valueOf(10)

什么才算“创建对象”？

在 Java 中，只有当在堆内存中分配了空间，并完成初始化后，才算是真正创建了一个对象。

常见的创建方式包括：

  • new 关键字（最常用）
  • 反射（如 Class.newInstance()）
  • 克隆（clone() 方法）
  • 反序列化（ObjectInputStream.readObject()）

而以下情况不一定创建新对象：

  • 基本类型变量（如 int a）
  • 未初始化的引用（如 Integer b;）
  • 自动装箱时使用缓存（如 Integer b = 100;，值在 -128~127 范围内）
  • 字符串字面量（如 String s = "hello"; —— 会创建对象，但放入字符串常量池，且可能复用）

特别说明：String s = "hello"; 确实会创建 String 对象（如果常量池中尚不存在），但它不是通过 new 创建的，而是由 JVM 在类加载阶段处理。

总结

  • int a;：基本类型，不是对象。
  • Integer b;：引用类型，声明了引用，但未创建对象。
  • 真正的对象创建必须经过 JVM 的五步流程，通常由 new 触发。
  • 理解“声明”与“创建”的区别，是掌握 Java 内存模型和面向对象机制的关键一步。
